Application Requirements ‚Äî Madrid Bus Companion

A simple bus navigation app with an educational twist. Find your bus route, discover your wait time, and learn about Madrid while you wait through location-based quizzes powered by linked DBpedia entities.

App Concept

Madrid Bus Companion helps users navigate Madrid's bus network while making wait times more engaging. Users enter their destination, see the nearest bus stop and route information, and pass the waiting time with an educational quiz game about nearby places. The app demonstrates linked data by using geographic proximity to query DBpedia for cultural/historical information about places near bus stops.

Core Value: Practical bus navigation + Educational entertainment + Demonstration of how geographic coordinates enable linking bus stops with local knowledge from DBpedia.

How It Works

1. Enter Destination

User types or selects their destination. App uses GPS location to find the nearest bus stop and determines the best bus route.

2. View Route & Wait Time

Interactive map centers on the nearest bus stop, showing the bus route, stop name, next bus arrival time, and estimated wait duration calculated from GTFS schedule data.

3. Play Location Quiz

While waiting, user can play a Kahoot-style quiz about the local area. Questions are dynamically fetched from DBpedia based on nearby places (landmarks, history, culture).

4. Learn & Discover

Each quiz answer shows linked data from DBpedia, teaching users about their surroundings and demonstrating the power of linked open data.

User Interface

Bus Navigation Screen

Interactive map displays user location, highlights the nearest bus stop, shows the bus route to take, and presents clear wait time information. Simple, clean interface focused on getting users where they need to go.

Location Quiz Screen

Kahoot-style quiz interface presents questions about the local area. Users select answers and learn interesting facts about nearby landmarks, history, and culture sourced from DBpedia.

Key Features

  • Real GTFS Data Integration: All bus information calculated from actual Madrid CRTM bus schedules
  • GPS-Based Stop Detection: Automatic nearest bus stop finding using geocoordinates from merged GTFS dataset
  • Route Visualization: Map shows bus route paths using shape geometry data
  • Schedule-Aware Calculation: Respects service calendars (weekday/weekend, exceptions) for accurate bus wait times
  • Runtime DBpedia Integration: Geographic coordinates enable dynamic SPARQL queries to DBpedia for nearby landmarks
  • Linked Data Demonstration: Shows how standardized geographic coordinates facilitate linking across distributed RDF datasets (GTFS + DBpedia)
  • Educational Content: Users learn about their surroundings while waiting for the bus
  • Mobile-First Design: Clean, simple interface optimized for on-the-go use
