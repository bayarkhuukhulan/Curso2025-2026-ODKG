Analysis of the Spanish Open Mobility Dataset (AS2-G4)

1. Dataset overview

Publisher / Rightsholder: Spanish Ministry of Transport and Sustainable Mobility (Ministerio de Transportes y Movilidad Sostenible — MITMS). citeturn1view0turn9search0

Dataset description: The dataset consists of daily, nationally-aggregated mobility indicators derived from anonymized mobile phone data. Released data include:

  • Origin–destination (OD) matrices with auxiliary attributes (distance, inferred activity such as home/work/frequent visit, demographic indicators: age buckets, gender, income bins).
  • Number of trips per person (grouped: 0,1,2,>2) with overnight stay zone and demographic counts.
  • Spatial tessellations / zone definitions (districts, municipalities, greater urban areas).

Source: pySpainMobility paper and project repositories. citeturn1view0turn9search0

1.1 Temporal coverage

Two main publication phases:

  • Phase v1: February 2020 — June 2021 (CDR-derived).
  • Phase v2: January 2022 — present (passive measurement, improved methodology).

These ranges are reported in the pySpainMobility paper. citeturn1view0

1.2 Population coverage and scale

The dataset accounts for over 13 million users (as reported by MITMS & pySpainMobility). Records are published daily (with ~4-day publication delay) at three spatial aggregation levels. citeturn1view0

1.3 File formats and schema

Official releases and community libraries (pySpainMobility, spanishoddata) package the data as compressed Parquet files and CSV derivatives. Main record types (OD, trips-per-person, zones) have structured tabular schemas with the following representative fields:

DatasetKey fields
OD matrixdate, origin_zone_id, destination_zone_id, trips, trips_km, distance_median, activity_origin, activity_dest, age_bucket, gender, income_bin
Trips per persondate, residence_zone_id, overnight_zone_id, trips_count_bucket, population_count_by_demo
Zoneszone_id, zone_name, zone_type (district/municipality/gau), geometry / centroid (when available)

2. Licensing analysis

The dataset is published by the Spanish Ministry and accompanied by a ministry-specific Licencia de datos abiertos (MITMA open data license). The Ministry license file is linked from the mobility open-data portal (named e.g. LICENCIA de datos abiertos del MITMA 20201203.pdf). citeturn10search0turn7search3

Because the authoritative license is the Ministry’s LDA, any derivative dataset generated by our group must be compatible with that license. The MITMA LDA is a government open-data licence that typically permits redistribution and reuse subject to attribution and the Ministry’s stated conditions (see the official LDA document). In practice, two approaches are reasonable for our generated Linked Data:

  1. Adopt the same MITMA LDA for the generated RDF/OWL examples (recommended if the derived triples are merely a direct republishing/transform of original MITMA content).
  2. If redistribution or combination with third-party data requires a common open license, choose ODbL (Open Database License) or CC BY 4.0 depending on whether dataset-level copyleft is desired (ODbL) or permissive attribution is sufficient (CC BY 4.0). If the original LDA imposes share‑alike / database-style restrictions, ODbL is the safest match.

Relevant sources: MITMA open-data listing and MITMA license reference. citeturn7search3turn10search0

3. Resource naming strategy (URI design)

We define a clear, REST-friendly URI strategy for ontology and data. Use the following base pattern (replace GROUPXX and ccsi923.github.io/Curso2025-2026-ODKG with the actual group and hosting domain when publishing): 

https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/

Concrete namespaces used in the ontology and examples:

  • ont: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/ontology/ — ontology (Turtle/OWL) documents.
  • data: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/data/ — individual dataset resources (OD matrices, trips aggregates).
  • zone: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/zones/ — zone identifiers (municipalities, districts, GAU).

Recommended URI patterns:

  • Ontology file: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/ontology/pyspainmobility-ontology.ttl
  • Class IRIs: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/ontology#ODMatrix
  • OD instance: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/data/od/2022-03-20/municipalities/od-2022-03-20-mad-101-bar-080-example
  • Zone: https://data.ccsi923.github.io/Curso2025-2026-ODKG/HandsOn/AS2-G4/zones/municipality/080 (use authoritative zone codes if available)

Use content negotiation (serve Turtle at .ttl and HTML landing pages at the same IRI) when publishing on the web. Include dcterms:license and dcterms:provenance metadata on every dataset resource.

4. Lightweight ontology (design summary)

We designed a compact ontology covering the main entities and properties needed to represent the MITMA mobility outputs:

  • Classes: ont:MobilityDataset, ont:ODMatrix, ont:TripCount, ont:Zone, ont:StudyArea, ont:DemographicGroup.
  • Key properties: ont:hasOrigin, ont:hasDestination, ont:trips, ont:tripsKm, ont:date, ont:zoneType, dcterms:license, dcterms:provenance.

Full OWL/Turtle ontology and a small example RDF instantiation (Turtle) are included in /ontology.

5. Example instantiation policy

The example RDF file intentionally contains a tiny synthetic / anonymized extract (not raw original records) to demonstrate the mapping from the tabular schema to RDF using the ontology. The example includes: one OD resource (municipalities, 2022-03-20) and two zone resources with demographic buckets.
